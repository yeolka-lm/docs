Краткое руководство по командам бота для персонала. Все команды вызываются через `/` в Discord. В таблицах параметров ✅ — обязательный параметр, ❌ — необязательный.

## Повышения

<AccordionGroup>
  <Accordion title="/promote — повысить сотрудника" icon="arrow-up">
    Повышение сотрудника на следующую должность.

    | Параметр | Описание | Обязательность |
    | --- | --- | --- |
    | `user` | Пользователь, которого повышают | ✅ Да |
    | `reason` | Причина повышения | ✅ Да |
    | `department` | Отдел | ✅ Да |

    ```text Пример
    /promote user:@Иванов reason:Отличная работа за месяц department:Discord
    ```
  </Accordion>

  <Accordion title="/promotion view — просмотр повышения" icon="magnifying-glass">
    Просмотр информации о конкретном повышении.

    | Параметр | Описание | Обязательность |
    | --- | --- | --- |
    | `id` | ID повышения из истории | ❌ Нет |

    ```text Пример
    /promotion view id:42
    ```
  </Accordion>

  <Accordion title="/promotions — история повышений" icon="clock-rotate-left">
    Просмотр истории всех повышений.

    ```text Пример
    /promotions
    ```
  </Accordion>
</AccordionGroup>

## Отпуска (LOA)

<AccordionGroup>
  <Accordion title="/loa request — запрос отпуска" icon="plane-departure">
    Запрос на отпуск.

    | Параметр | Описание | Обязательность |
    | --- | --- | --- |
    | `start_date` | Дата начала отпуска | ✅ Да |
    | `end_date` | Дата окончания отпуска | ✅ Да |
    | `reason` | Причина отпуска | ✅ Да |

    ```text Пример
    /loa request start_date:15.07.2026 end_date:30.07.2026 reason:Личные дела
    ```
  </Accordion>

  <Accordion title="/loa admin — управление статусом LOA" icon="user-shield">
    Управление статусом LOA (только для руководства).

    | Параметр | Описание | Обязательность |
    | --- | --- | --- |
    | `user` | Сотрудник | ✅ Да |
    | `action` | `approve` / `deny` / `cancel` | ✅ Да |
    | `reason` | Причина решения | ❌ Нет |

    ```text Пример
    /loa admin user:@Петров action:approve reason:Одобрено
    ```
  </Accordion>

  <Accordion title="/loa manage — управление своим отпуском" icon="gear">
    Управление своим отпуском.

    | Параметр | Описание | Обязательность |
    | --- | --- | --- |
    | `action` | `extend` (продлить) / `cancel` (отменить) | ✅ Да |
    | `reason` | Причина | ❌ Нет |

    ```text Пример
    /loa manage action:extend reason:Нужно больше времени
    ```
  </Accordion>

  <Accordion title="/loa active — активные отпуска" icon="list-check">
    Просмотр всех активных отпусков.

    ```text Пример
    /loa active
    ```
  </Accordion>

  <Accordion title="/loa history — история отпусков" icon="clock-rotate-left">
    Просмотр истории отпусков.

    ```text Пример
    /loa history
    ```
  </Accordion>

  <Accordion title="/loa pending — ожидающие отпуска" icon="hourglass-half">
    Просмотр всех ожидающих подтверждения отпусков (только для руководства).


## Нарушения (Infractions)

Модуль нарушений позволяет документировать и отслеживать нарушения правил персоналом. Все нарушения централизованно хранятся в одном месте.

### Типы нарушений

| Тип | Действие |
| --- | --- |
| Add Role | Выдача роли нарушителю |
| Remove Role | Снятие роли у нарушителя |
| Channel | Отправка лога в другой канал |
| Removal From Staff Database | Удаление из базы персонала |
| Remove From Group | Удаление из группы Roblox (если используется) |
| Change Group Role | Изменение роли в группе Roblox |

### Команды

<AccordionGroup>
  <Accordion title="/infraction issue — выдать нарушение" icon="gavel">
    Выдача нарушения сотруднику.

    | Параметр | Описание | Обязательность |
    | --- | --- | --- |
    | `user` | Нарушитель | ✅ Да |
    | `type` | Тип нарушения | ✅ Да |
    | `reason` | Причина | ✅ Да |
    | `expiration` | Срок действия (в днях) | ❌ Нет |

    ```text Пример
    /infraction issue user:@Сидоров type:Remove Role reason:Злоупотребление полномочиями expiration:30
    ```
  </Accordion>

  <Accordion title="/infraction view — просмотр нарушения" icon="magnifying-glass">
    Просмотр и управление нарушением.

    | Параметр | Описание | Обязательность |
    | --- | --- | --- |
    | `id` | ID нарушения | ✅ Да |

    ```text Пример
    /infraction view id:15
    ```
  </Accordion>

  <Accordion title="/infraction list — список нарушений" icon="list">

    | Параметр | Описание | Обязательность |

    | `user` | Сотрудник | ❌ Нет |
    | `server` | Весь сервер | ❌ Нет |

  <Accordion title="/infraction multiple — выдать нарушение нескольким" icon="users">

    | Параметр | Описание | Обязательность |
    | --- | --- | --- |
    | `users` | Список пользователей | ✅ Да |
    | `type` | Тип нарушения | ✅ Да |
    | `reason` | Причина | ✅ Да |

    ```text Пример
    /infraction multiple users:@Сидоров @Петров @Иванов type:Remove Role reason:Неявка на собрание
    ```
  </Accordion>
